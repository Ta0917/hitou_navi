# Agent Checklist

## Start Of Session

- [ ] Read `SESSION_STATE.md`.
- [ ] Read `DECISIONS.md`.
- [ ] Read the latest entries in `WORKLOG.md`.
- [ ] Read `HANDOFF.md`.
- [ ] Check Git status if the target project is a Git repository.
- [ ] Confirm whether another agent is marked as current owner.
- [ ] Add a short "started work" entry to `WORKLOG.md` for substantial tasks.

## During Work

- [ ] Record durable decisions in `DECISIONS.md`.
- [ ] Keep `SESSION_STATE.md` accurate when owner, status, touched files, or next action changes.
- [ ] Avoid changing files marked as actively owned by the other agent unless the handoff says to continue them.
- [ ] If a conflict appears, stop and document it in `HANDOFF.md`.

## End Of Session

- [ ] Append a `WORKLOG.md` entry with changed files and verification.
- [ ] Update `SESSION_STATE.md`.
- [ ] Update `DECISIONS.md` if decisions were made or revised.
- [ ] Update `HANDOFF.md` with the next recommended action.
- [ ] Include test results, commands run, and any unverified assumptions.
